# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 在本仓库中工作时提供指导。

## 仓库概述

这是一个个人知识库和技术笔记仓库 (`note-md`)，采用 markdown 文档系统组织。

## 命令

### 构建和发布

将 markdown 文件转换为 HTML 并导出到配套仓库：
```bash
make default
```

构建并推送更改到两个仓库：
```bash
make push
```

`markhtml` 工具将所有 markdown 文件转换为 HTML，输出到 `../note/` 目录用于网页发布。

## 内容结构和元数据

### 文件组织

所有内容文件使用 markdown 格式，在 HTML 注释中包含结构化元数据：

```markdown
<!--
markmeta_author: [作者]
markmeta_date: [YYYY-MM-DD]
markmeta_title: [标题]
markmeta_categories: [分类]
markmeta_tags: [逗号分隔的标签]
-->
```

### 目录结构

每个分类目录包含：
- `index.md` - 目录，包含该分类下所有笔记的带日期链接
- 各主题的 markdown 文件
- `images/` 子目录（如需要）用于存放配图

主要分类：
- `ai/` - AI/ML 主题，如包含 prompts (`ai/prompts/`) 和开发指南 (`ai/ai-dev/`) 子目录
- `go/` - Go 编程语言深度解析
- `think/` - 战略思考和哲理文章
- `cloudnative/`、`db/`、`mq/`、`tools/` - 基础设施和工具
- `business/`、`methodology/`、`okr/` - 职业发展
- `python/`、`rust/` - 其他编程语言

## 写作风格和内容指南

在本仓库创建或修改内容时，遵循以下既定模式：

### 语言和语气

**双语技术写作**：
- 主要语言：中文
- 技术术语保持英文（如 "Transformer"、"GPT"、"goroutine"）
- 引入关键概念时在括号中提供英文翻译
- 代码示例和命令行指令始终使用英文

**语气**：
- 客观专业
- 实用导向
- 教育性但简洁
- 避免营销语言或过度热情

### 结构和格式

**内容组织**：
1. 从概念介绍和背景开始
2. 从基础到进阶递进
3. 包含实际示例和代码片段
4. 以参考资料和外部资源结尾

**标题**：
- 使用清晰的层级结构（##、###、####）
```
## 1. 一级标题1
### 1.1 二级标题1
### 1.2 二级标题2
## 2. 一级标题2
```
- 保持标题描述性和可操作性
- 展示顺序步骤时对主要章节编号

**列表和表格**：
- 无序信息使用项目符号
- 顺序步骤使用编号列表
- 表格用于比较选项或总结特性
- 代码块带语言标签以实现语法高亮

### 叙述方式

**教学风格**：
- 介绍概念时先解释"为什么"再解释"怎么做"
- 从基础知识逐步构建
- 适当使用类比和对比
- 包含主要技术的历史背景

**实践导向**：
- 优先展示可运行的示例而非纯理论
- 展示完整可运行的代码片段
- 包含设置和安装说明
- 提供故障排除技巧和常见陷阱

**引用和归属**：
- 在文末包含"## 参考"或"## A. Reference"章节
- 链接到权威来源（官方文档、研究论文、技术博客）
- 格式：带描述文字和 URL 的编号列表或项目符号
- 标注原作者和来源

### 内容示例

**技术教程**: 
- 包含操作系统特定变体的安装步骤
- 带完整代码块的配置示例
- 渐进式功能说明（基础 → 进阶）
- 技巧和最佳实践章节
- 可选的个人感受章节"## A. 感受"

**概念解释**:
- 历史发展时间线
- 核心概念分解，包含中英文术语
- 带类比的架构概述
- 相关学术论文链接
- 学习资源参考章节

**最佳实践集合**:
- 简洁的项目符号配具体建议
- 需避免的代码反模式
- 性能优化技巧
- 权威来源链接

## 常用模式

### 代码块格式
```bash
# 始终包含语言标识符
# 使用注释解释不明显的步骤
command --flag value
```

### 章节组织
1. **安装/设置** - 如何开始
2. **基本用法** - 核心功能和常见任务
3. **进阶功能** - 可选的深入探讨
4. **最佳实践** - 建议和模式
5. **参考资料** - 外部链接和引用

### 索引文件
每个分类的 `index.md` 应列出所有文档，包含：
- `[YYYY-MM-DD]` 格式的日期
- 带链接的标题
- 按时间倒序排列（大多数分类中最新的在前）

## 仓库约定

- **不添加自动生成内容警告**：不要添加关于 AI 生成内容的免责声明
- **保留元数据**：始终维护 markmeta 注释块
- **一致的命名**：文件名使用小写字母加连字符（如 `go-best-practices.md`）
- **图片引用**：将图片存储在分类级别的 `images/` 目录中
- **交叉引用**：引用其他文档时使用相对 markdown 链接

## 编写新内容

被要求创建新文档时：

1. **严格遵循元数据模板**，如上所示
2. **匹配该分类的现有语气和结构**
3. **优先包含实际示例**而非纯理论
4. **提供参考资料**链接到权威来源
5. **使用双语方式** - 中文叙述，英文技术术语
6. **添加到分类的 index.md**，包含正确的日期和标题
7. **保持简洁** - 这是参考集合，不是教科书

## Git 工作流

本仓库使用简单的提交信息：
- 近期模式：日常更改使用单词 "update"
- 提交频繁且增量式
- 更改同时推送到 `note-md`（源码）和 `../note/`（渲染的 HTML）
